🗺️ GeoSG - User Manual

User guide for using the GeoSG web geoportal for efficient spatial data management.

Welcome to the GeoSG Geoportal

The GeoSG geoportal allows viewing, visualization and management of spatial data such as cadastral parcels, buildings, roads and addresses.

The main features of the geoportal are divided into individual tabs accessible through the navigation menu at the top of the page. Select the desired tab and follow the instructions for each feature.

System Requirements for Using the GeoSG Geoportal

  • a modern web browser (Chrome, Firefox, Edge, Safari),
  • minimum screen resolution of 1280 × 720 px,
  • an active internet connection,
  • a user account with appropriate access rights (for data editing).

Geoportal Usage Basics

Main Parts of the Geoportal User Interface

The geoportal user interface consists of five main parts:

1

Navigation Menu


The navigation menu allows switching between individual geoportal modules. Clicking on a tab displays the corresponding table and form for data entry.

On the right side of the navigation menu, additional settings are available where you can change the geoportal language (SI, EN, ESP) and select the colour theme of the user interface. In addition to the default light theme, an alternative colour theme and a dark mode are also available.

2

Data Entry and Editing Form

The form allows entering new spatial objects with their attribute data and editing existing objects.

3

Map View

The interactive map allows data review, object selection and drawing and editing geometry.

4

Tabular Data View

The table displays a list of records and enables viewing, searching and selecting data for further work.

5

Status Bar

The status bar displays the current state of the geoportal and its responses to user commands. A detailed description is available in the "Advanced Features" section.

💡 Adjusting Panels You can adjust the size of individual interface sections by dragging the dividers between them. Grab the divider with the mouse and drag it in the desired direction. Clicking the arrow completely closes or reopens a panel.

The vertical divider changes the width of the form and map view, while the horizontal divider changes the height of the upper interface section and the tabular view.

Layer Display

On the map view (top right), a "Layers" button is available for managing the visibility of spatial layers.
Individual layers can be turned on and off using checkboxes, and you can also manage layer groups (e.g. WMS GURS, WFS GURS, custom and working layers). For a layer to be displayed, both the layer and its parent group must be selected.

A detailed explanation of individual layers is available in the "Layer Management" tab.


Layer Editing

The geoportal navigation menu tabs (Parcels, Buildings, Roads, Addresses) enable working with the corresponding types of spatial data.
Buttons for adding, selecting and editing geometries are located on the left side of the map view. They are active according to the currently selected navigation menu tab, meaning you always work with data from the selected module. Detailed use of features is presented separately for individual modules in the following tabs. Working with different types of spatial data is based on the same principles, so reviewing one example (e.g. parcels) is sufficient for understanding. Individual modules differ mainly in their attributes, which are presented at the beginning of each chapter.

Working with Parcels

Displaying Parcel Data in the Table

1

Select the "Parcels" tab in the geoportal navigation menu

All parcels from the database are displayed in the table with columns: ID, Parc. no., CM, Area, Geom WKT.

2

Selecting the Data Display Mode in the Table

Two selection buttons are available above the table:

  • Select all: Shows all parcels
  • Select one: Shows only the parcel with a specific ID (enter the ID in the field)
Search Parcel by Attributes Use the search function: enter the CM code and Parcel number, then click "Search".
The table will automatically scroll to the found parcel.

Adding a New Parcel

1

Draw Geometry on the Map

The parcels tab must be selected. Then on the left side of the map view, select the icon for drawing a parcel. When hovering over the icon, a tooltip appears with the description: "Start drawing parcel". Draw the polygon by clicking successively on the map view; end the capture with a double-click.

2

Fill in the Form

The WKT geometry is automatically transferred to the form (field "Geom"). You may modify it if desired. The remaining fields are required:

  • Parcel number (parc_st): e.g. "1234/5"
  • CM code (sifko): e.g. "850"

Area calculation is performed automatically based on the geometry and cannot be changed.

3

Save the Parcel

Click the Insert button. With appropriate user permissions, the parcel is saved to the database and shown in the table. It also becomes part of the vector layer displaying this type of spatial data.

Editing an Existing Parcel

1

Select a Parcel

You can select a parcel in two ways: in the table by clicking the Uredi or directly on the map using the tool "Select parcel". Regardless of the method chosen, the parcel data is loaded into the form.

2

Edit Attributes or Geometry

Attributes: Change the values in the form (parcel number or CM code).

Geometry: Change the coordinates of points in the geometry record.

Editing geometry directly on the map: Izberite ikono Edit parcel. You can then drag the vertices of the existing parcel to change its shape. When done, click the icon to disable the editing function. The WKT is updated within the form on the left.

3

Save Changes

To update the geometry in the database, click the action button: Update.
Changes to the parcel are saved to the database.

Deleting a Parcel

Find the parcel you want to delete in the table.
At the far right of the table in the "Action" column, select the action button: Delete.
In the confirmation popup window, confirm the deletion of the parcel.

Permissions To save, edit and delete, you must be logged in and have the role of editor or admin.
If you do not have the necessary permissions, a warning will appear and the operation will not be carried out.

Layers and Layer Management

The "Layers" control in the upper right corner of the map view enables turning individual spatial layers and layer groups on and off. For a layer to be displayed, both the layer and its parent group must be checked. When the application starts by default, all individual layers are enabled, but only the last layer group is active. Therefore, when entering the application, only the working layers are visible. The image below shows this initial state and an overview of all available layers.


Enabling and Managing Layers (Step by Step)

1

Open the Control

On the map (right panel), find the button "Sloji" in the upper right corner. Click it to open the layers menu.

2

Enable or Disable a Layer

Click the checkbox next to the name of an individual layer or layer group. A layer will be displayed on the map only if both the individual layer and its group are active simultaneously.

An Individual Layer Can Have Three Different States

  • Disabled layer: the layer is not selected and therefore not shown on the map, even if its group is active.
  • Ready layer: the layer is selected, but its group is not active, so the layer is not yet shown on the map.
  • Active layer: the layer is selected and belongs to an active group, so it is shown on the map.
⚠️ Important If an individual layer is selected but its group is not active, the layer will not be displayed. Likewise, an active group alone does not mean all layers within it will be displayed — each individual layer must also be checked.

Layer Types

GeoSG uses several types of layers, which differ in display method, data volume and intended use.

1

WMS GURS (Web Map Service)

WMS (Web Map Service) is an OGC (Open Geospatial Consortium) standard for accessing cartographic data as images over the web. WMS layers are displayed as raster images (e.g. PNG or JPEG), enabling fast display of larger spatial areas but not allowing direct access to geometry or attributes. The GeoSG application provides selected WMS layers from the Surveying and Mapping Authority of the Republic of Slovenia (GURS), accessed via links published on the eProstor portal. Available layers are:

  • Parcels: display of cadastral parcels
  • Buildings: display of the building cadastre
  • DOF25: digital orthophoto (spatial resolution 25 cm)
💡 Purpose and Use of WMS GURS Layers WMS GURS layers are used as reference layers for quick visual review of official spatial data and as a cartographic base when working in the application.
2

WFS GURS (Web Feature Service)

WFS (Web Feature Service) is an OGC (Open Geospatial Consortium) standard for accessing vector spatial data over the web. It enables the transfer of geometry and associated attributes, allowing their analysis and further processing. In the GeoSG application, WFS layers from GURS are used as reference data, enabling review of individual spatial objects (entities) together with their attributes. The links to this data were obtained through the eProstor portal.

Available layers are:

  • Cadastral municipalities: boundaries of cadastral municipalities (polygons)
  • Parcels: cadastral parcels with attributes (parcel number, area, CM etc.)
  • Buildings: buildings with attribute data
  • Municipalities (RPE): municipal boundaries from the register of spatial units
💡 Purpose and Use of WFS GURS Layers WFS GURS layers are used for a detailed review of individual spatial objects (entities) and their attributes and for data verification compared to own layers. In the application, they are intended exclusively for display and not for editing, as editing is enabled on the working (vector) layers.
⚠️ Important about WFS Layers WFS layers transfer A LOT of data (all geometries and attributes). This can significantly slow down the application, especially for large areas. We recommend using them only for smaller areas.
3

Custom WMS Layers (GeoServer)

Custom WMS layers: These are layers served by GeoServer from the PostgreSQL/PostGIS database of the GeoSG portal. They represent a raster display of data dynamically generated on the server. They may show a different state than GURS data, as the source data may have been edited, added or deleted.

  • Parcels: parcel data from your database
  • Buildings: building data
  • Roads: road data
  • Addresses: address data

The WMS service is also directly accessible via GeoServer, allowing layers to be included in external GIS tools (e.g. QGIS): http://64.225.137.73:8080/geoserver/wms?service=WMS&request=GetCapabilities

The same layers are also available via the WFS service, which enables access to vector data and its further processing: http://64.225.137.73:8080/geoserver/wfs?service=WFS&request=GetCapabilities

💡 Purpose and Use of Custom WMS Layers Custom WMS layers enable a quick visual overview of data from the GeoSG portal database, without transferring geometry. They are suitable for displaying larger areas or as reference layers when attribute information is not needed.
4

Working Layers (Vector)

Vector layers from the GeoSG database: These are layers loaded directly from the GeoSG geoportal's PostgreSQL/PostGIS database as vector data. They contain geometry and associated attributes, enabling interaction with objects. Working vector layers represent the basic working level of the application, as they enable direct data manipulation.

  • Parcels: enables selection, editing and capture of new parcels
  • Buildings: enables selection and editing of buildings
  • Roads: enables selection and editing of roads
  • Addresses: enables selection and editing of addresses
💡 Purpose and Use of Working Vector Layers They are used for selecting objects, editing geometry, updating attributes and capturing new data. All active data operations in the application take place on these layers.

⚠️ Performance
  • If the application slows down, first disable the WFS GURS layers
  • On a slow connection, use only WMS layers for reference display
  • For editing geometry and drawing, you must have the vector working layer you want to modify enabled

Layer Order

Layers are displayed in the following order (from bottom to top):

  1. Base map (OSM, Google satellite…) – at the bottom
  2. WMS GURS (GURS reference layers)
  3. WFS GURS (GURS vector layers)
  4. My WMS layers (from GeoServer)
  5. My vector layers (from the database)
  6. Imported layers (GeoPackage, Shapefile, KML)
  7. Highlight layer (selected objects - magenta outline) – on top

Advanced Features

Adding Custom Layers (GeoPackage, Shapefile, KML)

The application allows importing custom spatial files as additional layers on the map. Supported formats are GeoPackage (.gpkg), Shapefile (.shp) and KML (.kml). Imported layers are displayed above existing layers and can be toggled on or off freely.

1

Open the "Imported Layers" Panel

The "Imported Layers" section is located in the upper right area of the map view.
Click on it, then in the opened window press the button to import a file.

2

Select a File to Import

Click the file selection button (folder icon). Supported formats:

FormatExtensionNote
GeoPackage.gpkgRecommended format — can contain multiple layers
Shapefile.shp + .dbf + .prjSelect only the .shp file
KML.kmlGoogle Earth format

Managing Imported Layers

After selecting a file, the layer is automatically loaded onto the map. In the "Imported Layers" section, the file name appears with management icons shown in the table below.

IkonaFunctionDescription
Visibility Toggle layer display on the map
Remove layer Permanently remove the layer from the application

After import, layers are automatically added to the layer list on the right side, displayed at the bottom in a separate group "Imported layers".

Coordinate System The application uses the coordinate system EPSG:3794 (D96/TM – the official coordinate system in Slovenia). Imported files are automatically converted to this system. If the file does not contain projection information (e.g. Shapefile without .prj), EPSG:4326 (WGS84) is assumed by default.
Shapefile — required files The Shapefile format requires at least three files: .shp (geometry), .dbf (attributes) and .prj (projection). All three must be present in the same folder. When selecting a file, choose the .shp file; the other two are read automatically.
Imported Layers Are Temporary Imported layers are visible only in the current browser session. After refreshing the page (F5), they are lost. For permanent data storage, use the CRUD function (database entry) or GeoPackage export.

Exporting Data to GeoPackage

To export all data (parcels, buildings, roads and addresses), press Alt + Shift + E. The application will automatically create a GeoPackage file (.gpkg) and download it to your downloads folder as: podatki.gpkg.


Snap Functionality (Automatic Snapping)

The snap functionality ensures that when drawing or editing geometry, vertices automatically snap to existing vertices or lines. When the cursor approaches an existing point or line, a blue dot appears on it, indicating that snap is active. By clicking, the new point is precisely aligned with the existing geometry.

Warning If snap is not used, inaccurate geometries may result (e.g. overlaps or gaps between objects).

Frequently Asked Questions (FAQ)

How can I change the geometry of an existing parcel?
In the table click Edit, then on the map click the "Edit parcel" icon. Drag the vertices to the new position. When done, click the icon again (it turns off). The WKT in the form is updated. Click Update.
Why is the "Insert" or "Delete" button not visible?
You do not have the necessary permissions. Check whether you are logged in and whether you are a member of the 'editors' or 'admin' group. Contact the administrator to be assigned the appropriate permissions.
How do I export data to QGIS?
Press Alt + Shift + E to export the GeoPackage (.gpkg) file. In QGIS, use Layer → Add Layer → Add Vector Layer.
Why is the map loading slowly?
You probably have the GURS WFS layers enabled, which load a lot of data. Disable WFS layers in the "Layers" control in the top right of the map.
How can I reset the form?
Click the Clear button. All fields are reset.
What does "POLYGON(...)" mean in the Geom field?
This is a WKT (Well-Known Text) geometry representation. The application generates it automatically.
How can I find a parcel by cadastral municipality and parcel number?
Enter the CM code and Parcel number and click Search.
Can I draw multiple objects at once?
No. Each object must be saved before proceeding to draw the next one.
How do we change the coordinate system?
The application uses EPSG:3794. Changing it is not possible in the user interface.
Where can I see whether an operation was successful?
In the status bar at the bottom of the screen.
How can I test the API?
Odprite http://[strežnik]/swagger/.
Why does "Data is being loaded" take a long time?
You are probably loading a lot of data (e.g. WFS layers). Disable them or reduce the area.
How do I report a bug or suggest an improvement?
Contact the developer and include a description and steps to reproduce.
